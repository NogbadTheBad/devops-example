Brief History

  • Software release cycles often measured in quarters or years
    • Detailed specifications for software written and approved before development starts
    • Called "waterfall" Software Development Life Cycle
  • Running applications required specialized hardware, operating systems
    • Operations, network, database, security, testing skills also specialized

Challenges

  • Releases were huge and hard to plan, test
  • Waterfall SDLC makes changes hard
  • Monolithic software makes testing hard
  • Technical Leads and Architects lay out solutions before code is written
    • Often excludes key learnings from on-going development and stiffles developer creativity
  • "Hand-off" from Lead → Developer → Tester → Release Engineer → Operations
    • Can leave big gaps

Modern Thinking

Confluence of advances

  • Smaller, incremental releases foster agility
  • Commodity server hardware, often in the cloud
  • Skills and responsibilities not so siloed
  • Component approach decouples services
    • Microservice is set of related functionality with its own isolated datastore
    • Very likely a RESTful API with JSON
  • Containers (Docker)
  • Platform-as-a-Service (AWS, Azure, Google) runs Docker containers at scale
